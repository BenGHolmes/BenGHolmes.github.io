WikiGame Bot

Personal Project (Nov 2018)

The WikiGame is a game where players race to find the fastest path between two random Wikipedia pages by following links in articles. To guarantee that I always beat my friends, I decided to build a bot to play using a graph of all connected Wikipedia pages, rather than guessing what the fastest path is myself.

I downloaded the latest Wikipedia XML dump (a 50GB xml file), pulled all the links from every Wikipedia page, and stored them in a SQL database for quick access during games. I then used Selenium to interact with the game itself and follow the links on the pages. When a game starts, the bot grabs the start and end pages, and does a breadth first search of the graph to find the 20 fastest paths between the two pages (you get extra points for repeated wins within the time limit). I then quickly follow the links, completing as many paths as possible within the two minute time limit. The bot regularly scores 5-10x higher than its human opponents, and you can find it at the very top of the scoreboard at thewikigame.com (the username is "TheBeast").

Ball Bouncing in a Ball Motion Simulation

Personal Project (Oct 2018)

Recently, I've been really interested in the field of computational physics. In order to get some experience solving computational problems, I decided to implement the simulation outlined in this paper by Tanner Morrison. The code was written in MATLAB and can be found on my github.

The code performs finite difference analysis to simulate the motion of both balls, and the result of any impacts that occur between them or the container around them. The weight of each ball and the initial position and velocity can all be set to vary the outcome. In the future, I hope to add in friction and rolling to make the model more accurate.

Autonomous Rescue Robot with Computer Vision

Course Project (June - Aug 2018)

During this past summer, I participated in the yearly Engineering Physics robot design course. The course is structured as a competition, with the goal being to build a fully autonomous robot that is able to complete the given challenge in just 13 weeks. This year, we had to rescue Ewoks (stuffed animals) from the Empire stronghold by navigating over gaps, differentiating between various infrared light signals, and detecting and capturing 5 stuffed Ewoks along the way.

From the beginning, our team decided that we wanted to try to implement a universal solution, that required no hardcoding or prior knowledge of the course. In order to achieve this, I implemented a computer vision system that allowed us to navigate directly from Ewok to Ewok, without needing to look for specific course landmarks. A Raspberry Pi on the robot continuously captured images of the course ahead using a fisheye camera, and these photos were processed using a Python implementation of the extremely fast YOLO network, running on the Pi. The raspberry pi would then calculate how much the robot needed to turn in order to face the next Ewok, and then pass this information to our main STM32 microcontroller, which controlled the motors.
